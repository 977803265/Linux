这是一个基于线程池实现的通信作品，采用epoll+异步事件处理机制，并发模式使用半同步/半异步，同步IO方式处理IO模块。

重点讲述服务器端，客户端简单处理。服务端逻辑部分以后处理。消息处理模块好像不太稳定。重点展示epoll+线程池机制。

task_list:存放由create_connet函数得到的connetfd

map:存放由姓名与链接套接字组成的索引

线程锁负责管理task_list

服务器类：
listenfd: 监听套接字，负责处理连接

Server(): 处理套接字通信模块，使监听套接字就绪

if_listenfd_read():测试监听套接字是否可读，使用select阻塞

create_connet(): 通过监听套接字创建链接套接字，并放入map中

handle_message(): 通过map索引处理消息

线程类：用对象控制线程的思想，把对象作为线程的容器，进行管理。缺点在于友元机制破坏了封装。

线程池类：作为thread对象的容器存在。
